# /van — Инициализация и входная точка (Memory Bank)

Ты — AI-агент в Cursor. Твоя задача в этом режиме: **инициализировать Memory Bank**, определить платформу, оценить сложность задачи и направить пользователя на следующий шаг.

## Входные данные
- Если пользователь НЕ указал описание задачи после `/van`, сначала задай 1 вопрос: **«Какую задачу делаем? Опиши одной фразой.»** и остановись.
- Если описание задачи есть — используй его как `Task`.

## Шаги
1. **Платформа**
   - Определи ОС (darwin/windows/linux) и выбери корректные команды для терминала.

2. **Проверка Memory Bank**
   - Убедись, что существует `memory-bank/`.
   - Убедись, что существуют файлы:
     - `memory-bank/tasks.md`
     - `memory-bank/activeContext.md`
     - `memory-bank/progress.md`
     - `memory-bank/projectbrief.md`
   - Если чего-то нет — создай минимальные шаблоны.

3. **Заполнить стартовое состояние**
   - Обнови `memory-bank/tasks.md`:
     - `Task`: текст задачи
     - `Complexity`: пока пусто или `TBD`
     - `Plan`: пусто
     - `Checklist`: минимум 1 пункт
   - Обнови `memory-bank/activeContext.md`: текущий фокус = задача.

4. **Оценка сложности (Level 1–4)**
   - Level 1: быстрый фикс, 1–2 файла, без дизайна/архитектуры.
   - Level 2: небольшое улучшение, несколько файлов, требования ясные.
   - Level 3: новая фича/существенная логика, нужны решения/варианты.
   - Level 4: архитектура/подсистемы, серьёзные риски.
   - Если не хватает данных — задай до 3 уточняющих вопросов.

5. **Маршрут**
   - Если Level 1 → скажи пользователю: **«Дальше запускай `/build`.»**
   - Если Level 2–4 → скажи: **«Дальше запускай `/plan`.»**

## Выход
- Коротко сообщи:
  - определённую сложность
  - какие файлы Memory Bank обновлены
  - какую команду запускать следующей
