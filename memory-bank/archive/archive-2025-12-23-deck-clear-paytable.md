# Archive: 2025-12-23 — Paytable “Deck Clear” 150.000x + MAX WIN format

## Summary
- В paytable добавлен отдельный информационный блок **DECK CLEAR** (на английском, без слова “jackpot”) с условием полной очистки 54‑картной колоды за один каскад и выплатой **150.000x**.
- Формат **150.000x** унифицирован в MAX WIN оверлеях (исправлены `150,000X` и случайный дубль строки).

## Requirements
- В `PaytableModal` добавить новый блок про особую выплату:
  - язык: английский;
  - **не использовать слово “jackpot”**;
  - условие: за один каскад на столе не осталось карт и в колоде 54 карт больше нет карт;
  - выплата: **150 000x** (в UI в формате **150.000x**, как в верхнем HUD);
  - оформить “красиво”, по фидбеку: центрирование и порядок “x/сумма сверху, описание снизу”, в конце текста “you get MAX WIN”.
- Привести отображение MAX WIN множителя к единому формату **150.000x**.

## Implementation
- `src/ui/screens/balatro-inferno/components/PaytableModal.jsx`
  - Добавлена константа `DECK_CLEAR_MULTIPLIER = 150000`.
  - Добавлен спец-блок “DECK CLEAR” перед списком комбинаций:
    - рассчитывает `deckClearPayout = bet * DECK_CLEAR_MULTIPLIER`;
    - рендерит 150.000x (с маленькой `x`) + деньги через `formatMoneyFull`;
    - текст центрирован и заканчивается акцентом “MAX WIN”.
- `src/ui/screens/balatro-inferno/BalatroInferno.jsx`
  - Унифицировано отображение MAX WIN: `150,000X` → `150.000x` (маленькая `x`).
  - Убран случайный дубль строки `150.000x` в одном из оверлеев.

## Testing
- `npm run build` — ✅

## Lessons learned
- Для UI-чисел (особенно “150.000x” vs “150,000X”) лучше держать единый “источник формата” (helper/константа), чтобы не ловить расхождения по проекту.
- После правок в нескольких местах важно быстро просматривать целевые блоки, чтобы не пропустить дубли/опечатки.
