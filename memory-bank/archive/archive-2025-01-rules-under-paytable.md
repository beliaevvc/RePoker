# Архив — Перемещение RULES под PAYTABLE (2025-01)

## Summary
Простой UI-фикс уровня Level 1: изменён layout кнопок PAYTABLE и RULES в компоненте `HeaderBar2` с горизонтального на вертикальное расположение. Кнопка RULES теперь расположена под PAYTABLE с минимальным расстоянием между ними.

## Requirements
- Переместить кнопку RULES под PAYTABLE (вертикальное расположение)
- Сделать кнопки визуально ближе друг к другу

## Implementation

### Изменения в коде
**Файл:** `src/ui/screens/balatro-inferno/BalatroInferno.jsx`

1. **Изменён layout контейнера (строка 181):**
   - Было: `flex items-center gap-3`
   - Стало: `flex flex-col items-start gap-0`
   - Направление изменено с горизонтального на вертикальное
   - Отступ между элементами убран (`gap-0`)

2. **Добавлен отрицательный отступ для кнопки RULES (строка 204):**
   - Добавлен класс `-mt-2` для визуального сближения с PAYTABLE
   - Отрицательный margin-top уменьшает расстояние между кнопками

### Технические детали
- Использован стандартный подход с `flex-col` для вертикального расположения
- Отрицательный margin применён для тонкой настройки визуального расстояния
- Сохранено выравнивание по левому краю (`items-start`) для согласованности с интерфейсом

## Testing
- ✅ Линтер: ошибок нет
- ✅ Визуальная проверка: кнопки расположены вертикально, RULES под PAYTABLE
- ✅ Функциональность: обе кнопки работают корректно (открывают соответствующие модальные окна)

## Lessons Learned

### Что сработало хорошо
- Простое решение через изменение направления flex-контейнера
- Использование отрицательного margin для визуальной настройки без изменения структуры
- Итеративный подход к настройке расстояния по запросу пользователя

### Что можно улучшить
- Для UI-задач с визуальной настройкой можно сразу предлагать несколько вариантов расстояния, чтобы сократить количество итераций
- Можно использовать отрицательные отступы сразу при планировании, если требуется минимальное расстояние

### Выводы
Задача демонстрирует важность итеративного подхода к визуальным изменениям, где финальный результат зависит от субъективного восприятия пользователя. Простые UI-фиксы требуют внимания к деталям визуального представления.
