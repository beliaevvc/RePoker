# Архив — Логотип в стиле Balatro Pokerfall

**Дата:** 2025-01-XX  
**Ветка:** `logo-pokerfall` → `main` (слита)  
**Статус:** ✅ Завершено

---

## Summary

Создан текстовый логотип "POKERFALL" в стиле Balatro Pokerfall с неоновыми эффектами, glitch-анимациями и миниатюрой джокера. Логотип заменяет предыдущий "RESIMPLE GAMES" и включает:

- Золотой градиент с shimmer-эффектом
- Неоновое свечение с пульсацией
- Автоматические и интерактивные glitch-эффекты
- Миниатюра джокера слева от текста с пиксельными мастями
- Полная адаптивность для всех разрешений экрана

---

## Requirements

### Исходные требования
- Создать текстовый логотип в стиле Balatro Pokerfall вместо "RESIMPLE GAMES"
- Сохранить интерактивность (hover/click burst)
- Сохранить адаптивность размеров
- Соответствовать пиксельному ретро-стилю проекта
- Не перегружать производительность

### Дополнительные требования (в процессе реализации)
- Добавить миниатюру джокера слева от текста
- Использовать пиксельные масти (как в основном компоненте Card)
- Применить пиксельный шрифт для букв
- Отображать джокера на всех разрешениях (включая мобильные)

## Implementation

### 1. Creative Phase
**Файл:** `memory-bank/creative/creative-logo-pokerfall.md`

**Решения:**
- Цветовая палитра: золотой градиент (`#FFD700` → `#FFA500` → `#FF6347`)
- Неоновое свечение: умеренное с пульсацией (3s цикл)
- Glitch-эффекты: автоматические каждые 2.8s (±2-3px), усиленные при burst (±4-5px)
- Shimmer: медленный в idle (5s цикл), быстрый при burst (2s)

### 2. CSS-эффекты
**Файл:** `src/balatroInferno.css`

**Добавлено:**
- `.logo-gradient-gold` — градиент с shimmer-анимацией (5s цикл)
- `.logo-neon-pulse` — пульсация неонового свечения (3s цикл)
- Усиленные glitch-эффекты (`logo-glitch-burst`, `logo-glitch-hit`)
- Обновлённые ghost-слои с RGB-сдвигами (розово-красный и голубой)
- Стили для мини-карт джокера (`.logo-mini-cards`, `.logo-mini-card.joker`)

**Ключевые особенности:**
- Все анимации используют `will-change` для оптимизации
- Поддержка `prefers-reduced-motion` для accessibility
- Адаптивные размеры через `clamp()`

### 3. Компонент ResimpleLogo
**Файл:** `src/ui/screens/balatro-inferno/BalatroInferno.jsx`

**Изменения:**
- Текст изменён: "RESIMPLE GAMES" → "POKERFALL"
- Применены классы `logo-gradient-gold` и `logo-neon-pulse`
- Сохранена интерактивность (hover/click burst)
- Сохранена адаптивность размеров

### 4. Миниатюра джокера
**Компонент:** `LogoPixelSuit` (внутри `BalatroInferno.jsx`)

**Реализация:**
- Создан компонент для пиксельных мастей и джокера
- Использованы те же SVG-пути, что и в основном компоненте `Card`
- Применён пиксельный шрифт `Press Start 2P` для букв
- Абсолютное позиционирование с отступом от текста (`left: -2em`)
- Отображается на всех разрешениях (включая мобильные)

**Цвета джокера:**
- Тело карты: градиент от `#FDB931` до `#D4AF37` (тёмнее золотой)
- Корона: `#FFE55C` (светлее золотой)
- Используются цвета из градиента `logo-gradient-gold` для согласованности

## Lessons Learned

### Удачные решения
1. **Переиспользование компонентов**: Использование существующего подхода для пиксельных мастей (`PixelSuit`) обеспечило единообразие стиля
2. **Единый градиент**: Использование одного золотого градиента для всего текста оказалось лучше разделения цветов
3. **Абсолютное позиционирование**: Для декоративного элемента рядом с текстом абсолютное позиционирование дало точный контроль без влияния на поток текста
4. **Итеративный подход**: Множественные итерации по позиционированию и цветам привели к лучшему результату

### Что можно улучшить
1. **Уточнение требований**: Стоит сразу уточнять детали позиционирования (абсолютное vs относительное) у пользователя
2. **Тестирование с пользователем**: Важно тестировать решения с пользователем сразу, а не предполагать требования (например, скрытие на мобильных было пересмотрено)

### Технические выводы
- Абсолютное позиционирование подходит для декоративных элементов, которые должны быть "привязаны" к тексту
- Важно использовать цвета из единой палитры для согласованности визуала
- Компактные элементы (мини-карты) не обязательно скрывать на мобильных, если они не перегружают интерфейс
