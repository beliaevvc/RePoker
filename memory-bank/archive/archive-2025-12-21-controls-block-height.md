# Archive — 2025-12-21 — Controls block height (TURBO/PLAY/AUTO/+/-)

## Summary
- Увеличена высота нижнего блока кнопок управления (TURBO / PLAY / AUTO / + / -), чтобы кнопки визуально стали “толще” и лучше читались.
- После первичной правки высота уменьшена ещё на ~10% по фидбэку, сохранив пропорции всех кнопок.

## Requirements
- Сделать блок кнопок заметно выше (ориентир: +50–60% по высоте).
- Сохранить **пропорциональность** по высоте для всех кнопок внутри блока.
- Не сломать mobile layout (без переполнений/сильного “съедания” зоны карт).
- Не менять игровую логику кнопок.

## Implementation
- Файл: `src/ui/screens/balatro-inferno/BalatroInferno.jsx`
  - На grid-контейнер контролов добавлен `min-height` через `clamp(...)`.
  - Усилен press-эффект кнопок (увеличены `border-b` и соответствующий `translate-y`).
  - Итоговая высота после твика “-10%”: `min-h-[clamp(97px,12.6vh,126px)]`.

## Testing
- `npm run lint` — ✅

- Примечание: в sandbox `vitest` падал на `EPERM (kill)`, поэтому гейты прогонялись вне sandbox.

## Lessons learned
- Когда элементы уже `h-full` и сетка `grid-rows-2`, самый надёжный способ масштабировать “толщину” — задавать `min-height` контейнеру (через `clamp`), а не править каждую кнопку отдельно.
- Полезно сразу считать целевые значения от базовой высоты из инспектора (px), чтобы быстрее попадать в ощущение “+60% → -10%”.
- В этом проекте стоит помнить про ограничения sandbox для `vitest` (EPERM).
