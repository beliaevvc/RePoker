# Активный контекст

## Текущий фокус
Аудит архитектуры RePoker (Clean Architecture), проверка структуры модулей/слоёв, поиск упрощений и оптимизаций производительности, подготовка пошагового плана маленьких изменений без поломок.

## Детали
- Фокус: карта зависимостей и границ слоёв, затем — список точечных улучшений/оптимизаций.
- Приоритет перфоманса: плавность/FPS и отзывчивость (особенно мобилки).
- Браузеры: Chrome (primary), затем Safari; есть iPhone для реальной проверки.
- Процесс: двигаемся маленькими шагами, перед каждым существенным изменением — вопрос “продолжаем?”, Memory Bank обновляем часто, чтобы другой агент мог подхватить.
- Важно по UX: **если после любого шага визуал/ощущения не нравятся — откатываем этот шаг целиком** и двигаемся дальше.
