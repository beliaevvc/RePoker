# Reflection — 2025-12-20 — MODE spacing (panel bottom air)

## Контекст задачи
Нужно было дать панели переключения режима **MODE: NORMAL/CASCADE** больше “воздуха” снизу, чтобы она визуально не “прилипала” к блоку **ANTE**.

## План vs факт
- **План**:
  - Найти разметку панели MODE в `src/ui/screens/balatro-inferno/**`.
  - Добавить вертикальный spacing между MODE и блоком ANTE (mobile-first).
  - Прогнать гейты: `npm run lint`, `npm run test`, `npm run build`.
- **Факт**:
  - Панель MODE найдена в `src/ui/screens/balatro-inferno/BalatroInferno.jsx`.
  - Добавлен нижний отступ контейнеру MODE: `mb-2 sm:mb-3`.
  - Гейты пройдены: `npm run lint`, `npm run test`, `npm run build`.

## Что сделали
- Добавили вертикальный “разделитель” (margin-bottom) под панелью MODE, чтобы следующий ряд (грид с CHIPS/ANTE) начинался ниже.

## Что было сложным / почему
- Логика задачи простая; единственный нюанс — **0px** фактического расстояния между блоками в DOM, из-за чего визуально казалось “склеенным”.

## Что сработало хорошо
- Точечная правка на уровне контейнера (`mb-*`) дала предсказуемый результат без влияния на layout грида.
- Mobile-first подбор (`mb-2 sm:mb-3`) даёт чуть больше воздуха на больших экранах, не раздувая вертикаль на мобилках.

## Что можно улучшить в процессе
- Для подобных UI-правок полезно держать небольшую “шкалу” стандартных вертикальных отступов (например, 8/12/16px) и придерживаться её, чтобы spacing в шапке экрана был более консистентным.
