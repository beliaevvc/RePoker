# Рефлексия — 2025-12-21 — Jackpot Sim button (QA) reposition

## Контекст
Нужно было переделать кнопку `JACKPOT SIM (QA)` в экране **BalatroInferno**: сделать её маленькой, еле заметной и расположить справа от кнопки **PLAY**.

## План vs факт
- **План**
  - Найти место в UI, где рендерятся `PLAY` и `JACKPOT SIM (QA)`.
  - Убрать широкую QA-кнопку и перенести рядом с PLAY (маленькая, low-emphasis).
  - Прогнать `npm run lint` и `npm run build`.
- **Факт**
  - Нашёл разметку в `src/ui/screens/balatro-inferno/BalatroInferno.jsx`.
  - Удалил широкую кнопку `JACKPOT SIM (QA)` (которая занимала всю ширину).
  - Добавил компактную кнопку “JP” в правую колонку рядом с PLAY (только в режиме `cascade`), с низкой базовой opacity и нормальной видимостью на hover.
  - `npm run lint` и `npm run build` прошли успешно.

## Что сделали (результат)
- QA-фича осталась доступной, но больше не “кричит” в UI.
- Визуально соответствует требованию: **маленькая, справа от PLAY, еле заметная**.

## Что было сложным / почему
- Локально `npm run lint` в sandbox упал с `EPERM` при попытке чтения файла из `node_modules`.
  - Решение: запуск линта/сборки вне песочницы (разрешение `all`).

## Удачные решения
- Перенос в правую колонку рядом с debug-кнопками — логично для QA/дев-инструмента: рядом, но не мешает основной CTA (PLAY).
- Low-emphasis через opacity + hover — сохраняет доступность, не перегружая интерфейс.

## Что можно улучшить в процессе
- Если будет больше QA-кнопок, стоит выделить единый “QA toolbar” компонент/слот и контролировать видимость через флаг (например, env/URL-параметр), чтобы QA-инструменты не жили в основной верстке.
